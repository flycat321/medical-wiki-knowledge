### 2.1 三大本质差异

| 对比维度 | 传统施工图设计 | EPC设计总承包 |
|---------|--------------|-------------|
| **服务边界** | 出图为主，配合为辅 | 出图只是起点——还包括优化、变更、施工配合、竣工验收 |
| **时间维度** | 线性流程（方案→初设→施工图→施工） | 并行流程（设计与施工交叉，基础施工时上部还在出图） |
| **责任界面** | 对甲方负责，专业间内部协调 | 对内协调各专业+对外协调总包/业主/分包三方 |

### 2.2 设计人员须转变的认知

| 传统思维 | EPC思维 |
|---------|--------|
| "我把图画完交出去就行" | "图交出去只是开始，还要跟到施工完" |
| "这个问题应该施工单位解决" | "设计阶段能解决的问题不留给施工阶段" |
| "按规范做就没问题" | "规范是底线，还要考虑可施工性和造价" |
| "变更是甲方的事" | "主动预判变更，在设计阶段消化" |
| "专项设计是分包的事" | "分包的图我也要审，出了问题一起担" |

### 2.3 EPC模式下增加的设计服务内容

| 增加的内容 | 说明 | 对应章节 |
|----------|------|---------|
| 设计优化 | 在满足功能的前提下降低造价 | 第八章 |
| 限额设计 | 按概算控制各专业造价 | 第六章 |
| 施工配合 | 施工图交底、现场答疑、变更处理 | 第三章/第八章 |
| 分包管控 | 审核各专项分包的设计成果 | 第五章 |
| 材料/设备选型配合 | 参与材料/设备的技术比选 | 第六章 |
| 竣工验收配合 | 配合竣工图编制和验收 | 第九章 |
