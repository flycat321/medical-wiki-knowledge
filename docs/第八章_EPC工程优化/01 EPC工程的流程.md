### 1.1 传统模式 vs EPC模式

```
传统模式（串行）：

  方案设计 → 初步设计 → 施工图设计 → 招标 → 施工 → 竣工验收
  ──────────────────────────────────────────────────────────────→ 时间轴
  │←──── 设计阶段 ────→│         │←──── 施工阶段 ────→│


EPC模式（并行）：

  方案设计 → 初步设计 ─┬─ 施工图设计（分批出图）──────────→
                        │
                        └─ 基础施工 → 主体施工 → 安装 → 装修 → 竣工
  ──────────────────────────────────────────────────────────────→ 时间轴
  │←── 设计 ──→│←────── 设计与施工交叉并行 ──────→│

  ★ 关键区别：
  1. 施工图分批出图（先出基础→再出主体→再出安装→最后出装修）
  2. 设计变更即时影响施工（没有"全部画完再施工"的缓冲期）
  3. 设计深度须倒排（施工需要什么图就先出什么图）
```

### 1.2 EPC工程典型阶段划分

| 阶段 | 时间参考（1000床） | 设计工作 | 施工工作 |
|------|-----------------|---------|---------|
| 前期 | 0-3个月 | 方案设计+报规 | 场地平整+临建 |
| 初设 | 3-6个月 | 初步设计+报批 | — |
| 基础 | 6-10个月 | 基础施工图 | 基坑支护+基础施工 |
| 主体 | 10-22个月 | 主体施工图（分批） | 主体施工 |
| 安装 | 16-28个月 | 安装施工图+各专项图 | 机电安装 |
| 装修 | 22-34个月 | 装修施工图+专项深化 | 室内外装修 |
| 调试 | 30-36个月 | 配合调试+竣工图 | 系统调试+竣工验收 |

> **★ 设计周期和施工周期大量重叠，这意味着：**
> - 基础施工图必须在初设完成后立即启动
> - 主体施工图须在基础施工期间完成
> - 设计变更的影响是即时的——图纸一改，施工现场就要调整
